## What Are Variables? 📦

A **variable** is a **named container** that holds a value in your program's memory.

Think of it like a **labeled box**:
```
┌─────────────┐
│  name = "Alex"  │   ← The box is labeled "name" and holds "Alex"
└─────────────┘

┌─────────────┐
│  score = 95  │   ← The box is labeled "score" and holds 95
└─────────────┘
```

### 🧭 Real-World Analogy: Parking Lot 🅿️

Imagine a parking lot:
- Each **parking space** has an **address** (like A1, A2, B1) — that's the **variable name**
- Each space can hold **one car** — that's the **value**
- You can **replace** the car in a space — that's **reassignment**
- The space **remembers** which car is parked there — that's **memory**

### Creating Variables

```python
name = "Alex"        # Store a name
age = 14             # Store an age
score = 95.5         # Store a score
is_student = True    # Store a yes/no answer
```

## Assignment & Naming Rules 📏

### The Assignment Operator `=`

The `=` sign in Python doesn't mean "equal" like in math. It means **"store this value in this variable"**.

```python
x = 5          # Store 5 in x
message = "Hi" # Store "Hi" in message
```

**Read it as:** "x **gets** 5" or "x **is assigned** 5"

### Variable Naming Rules ✅❌

| Rule | Valid ✅ | Invalid ❌ |
|------|---------|-----------|
| Must start with letter or `_` | `my_var`, `_count` | `2cool`, `@name` |
| Can contain letters, numbers, `_` | `score1`, `player_2` | `my-var`, `my var` |
| Cannot be a Python keyword | `my_class`, `count` | `if`, `for`, `print` |
| Case sensitive | `Name` ≠ `name` | — |

### Good Naming Practices 🎯

```python
# ✅ GOOD — descriptive names
student_name = "Alex"
total_score = 95
is_passing = True

# ❌ BAD — vague names
x = "Alex"
n = 95
b = True
```

## Reassignment & Tracking Change 🔄

Variables can be **overwritten** — the old value is replaced by the new one.

```python
score = 10
print(score)    # 10

score = 25
print(score)    # 25 (old value 10 is gone!)

score = score + 5
print(score)    # 30 (25 + 5)
```

### 🧭 RPG Backpack Analogy 🎒

Imagine your character has a backpack with limited slots:
- Slot 1: Sword → you find a Shield → must **drop Sword to pick up Shield**
- That's reassignment! The old value is **gone forever** unless you saved it somewhere else.

### The Pattern: `x = x + 1`

This looks weird in math, but in programming it means: **"take the current value of x, add 1, and store it back"**.

```python
counter = 0
counter = counter + 1   # counter is now 1
counter = counter + 1   # counter is now 2
counter = counter + 1   # counter is now 3
print(counter)           # 3
```

### Shorthand Operators

```python
score = 10
score += 5    # Same as: score = score + 5  → 15
score -= 3    # Same as: score = score - 3  → 12
score *= 2    # Same as: score = score * 2  → 24
score //= 4   # Same as: score = score // 4 → 6
```

## Variable Swapping 🔀

How do you swap the values of two variables?

### ❌ The Wrong Way
```python
a = "apple"
b = "banana"
a = b        # a is now "banana"
b = a        # b is also "banana"! We lost "apple"!
```

### ✅ The Right Way — Using a Temporary Variable
```python
a = "apple"
b = "banana"

temp = a      # temp = "apple"
a = b         # a = "banana"
b = temp      # b = "apple"

print(a)      # banana
print(b)      # apple ✅ Swapped!
```

### ✅ Python Shortcut
```python
a = "apple"
b = "banana"
a, b = b, a   # Python magic swap!
print(a)       # banana
print(b)       # apple
```

## Variable Tracing 🔍

**Tracing** means following your code line by line and tracking what each variable holds at each step.

### Example: Trace This Code

```python
x = 3
y = 7
x = x + y
y = x - y
x = x - y
```

| Line | `x` | `y` | What happened |
|------|-----|-----|---------------|
| `x = 3` | 3 | — | x gets 3 |
| `y = 7` | 3 | 7 | y gets 7 |
| `x = x + y` | 10 | 7 | x gets 3+7=10 |
| `y = x - y` | 10 | 3 | y gets 10-7=3 |
| `x = x - y` | 7 | 3 | x gets 10-3=7 |

**Result:** x and y got swapped without a temp variable! 🤯
