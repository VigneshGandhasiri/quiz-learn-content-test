# ⚙️ Functions — Reusable Code Machines

Functions let you **wrap up code** into reusable, named blocks. Instead of copying the same code everywhere, you call a function by name! Functions are one of the most powerful tools in programming.

## Why Functions? 🤔

Without functions:
```python
# Calculate area of rectangle 1
length1, width1 = 5, 3
area1 = length1 * width1
print(f"Area: {area1}")

# Calculate area of rectangle 2 (SAME code again!)
length2, width2 = 7, 4
area2 = length2 * width2
print(f"Area: {area2}")
```

With functions:
```python
def calculate_area(length, width):
    return length * width

print(f"Area: {calculate_area(5, 3)}")   # 15
print(f"Area: {calculate_area(7, 4)}")   # 28
```

## Defining & Calling Functions 📝

```python
# DEFINE a function (create the recipe)
def greet():
    print("Hello!")
    print("Welcome to Python!")

# CALL the function (use the recipe)
greet()
greet()   # Can call it as many times as you want!
```

### Structure:
```python
def function_name():     # def keyword + name + parentheses + colon
    # indented code      # This is the function BODY
    # runs when called
```

## Parameters & Arguments 📨

**Parameters** are variables that functions accept as input.

```python
def greet(name):                    # 'name' is a parameter
    print(f"Hello, {name}!")

greet("Alex")                       # "Alex" is an argument
greet("Sam")
```

### Multiple Parameters
```python
def introduce(name, age, hobby):
    print(f"I'm {name}, {age} years old.")
    print(f"I love {hobby}!")

introduce("Alex", 14, "coding")
introduce("Sam", 12, "drawing")
```

### Default Parameters
```python
def power(base, exponent=2):    # Default exponent is 2
    return base ** exponent

print(power(5))       # 25 (5²)
print(power(5, 3))    # 125 (5³)
```

## Return Values 📤

Functions can **send back a result** using `return`.

```python
def add(a, b):
    return a + b

result = add(3, 7)
print(result)          # 10

# Use directly in expressions
total = add(10, 20) + add(30, 40)
print(total)           # 100
```

### Return vs Print
```python
# print() just displays — you can't use the result!
def add_print(a, b):
    print(a + b)

# return gives back a value you can use!
def add_return(a, b):
    return a + b

x = add_print(3, 4)    # Prints 7, but x = None!
y = add_return(3, 4)    # Nothing printed, but y = 7!
```

### Multiple Return Values
```python
def min_max(numbers):
    return min(numbers), max(numbers)

lowest, highest = min_max([3, 1, 7, 2, 9])
print(f"Min: {lowest}, Max: {highest}")  # Min: 1, Max: 9
```

## Scope 🔭

**Scope** = where a variable can be seen and used.

```python
def my_function():
    local_var = "I'm LOCAL"     # Only exists inside this function
    print(local_var)            # ✅ Works

my_function()
# print(local_var)              # ❌ NameError! Can't see it outside!

global_var = "I'm GLOBAL"       # Exists everywhere

def another_function():
    print(global_var)           # ✅ Can see global variables

another_function()
```

### Key Rule:
- Variables created **inside** a function are **local** — they disappear when the function ends
- Variables created **outside** all functions are **global** — visible everywhere

## Lists Inside Functions 📋

Functions can receive, modify, and return lists:

```python
def double_all(numbers):
    result = []
    for num in numbers:
        result.append(num * 2)
    return result

original = [1, 2, 3, 4, 5]
doubled = double_all(original)
print(doubled)    # [2, 4, 6, 8, 10]
```

### ⚠️ Lists are passed by reference!
```python
def add_item(lst, item):
    lst.append(item)     # This MODIFIES the original list!

my_list = [1, 2, 3]
add_item(my_list, 4)
print(my_list)           # [1, 2, 3, 4] — Changed!
```
